## Your First Translation

Let's translate a document in three simple steps:

<Steps>
  <Step title="Set Your API Key">
    Choose your preferred LLM provider and set the API key:
    
    ```bash
    export OPENAI_API_KEY="your-api-key-here"
    ```
    
    <Info>
      Don't have an API key? See the [Installation guide](/installation#api-keys-setup) for details on getting one.
    </Info>
  </Step>
  
  <Step title="Run Your First Translation">
    Translate a text document to German:
    
    ```bash
    tinbox translate --to de --model openai:gpt-5-2025-08-07 document.txt
    ```
    
    The translation will be printed to your console.
  </Step>
  
  <Step title="Save to a File">
    Add the `--output` flag to save the result:
    
    ```bash
    tinbox translate --to de --model openai:gpt-5-2025-08-07 --output document_de.txt document.txt
    ```
  </Step>
</Steps>

## Basic Examples

### Text Documents

Translating text files is the simplest use case:

<CodeGroup>
  ```bash Simple Translation
  tinbox translate --to es --model openai:gpt-5-2025-08-07 story.txt
  ```
  
  ```bash Specify Source Language
  tinbox translate --from zh --to en --model openai:gpt-5-2025-08-07 document.txt
  ```
  
  ```bash With Output File
  tinbox translate --to fr --model openai:gpt-5-2025-08-07 --output story_fr.txt story.txt
  ```
</CodeGroup>

### PDF Documents

<Warning>
  PDF translation requires:
  1. **Poppler** installed on your system (see [Installation](/installation#poppler-required-for-pdf-processing))
  2. A **vision-capable model** like GPT-4o, Claude Sonnet, or Gemini Pro Vision
</Warning>

<CodeGroup>
  ```bash Basic PDF Translation
  tinbox translate --to de --algorithm page --model openai:gpt-4o document.pdf
  ```
  
  ```bash With Custom DPI
  tinbox translate --to es --algorithm page --pdf-dpi 300 --model openai:gpt-4o document.pdf
  ```
  
  ```bash Save Output
  tinbox translate --to fr --algorithm page --output document_fr.txt --model openai:gpt-4o document.pdf
  ```
</CodeGroup>

<Tip>
  Use `--algorithm page` for PDFs to process them page-by-page, which works best for maintaining formatting and context.
</Tip>

### Word Documents (DOCX)

```bash
# Translate a Word document
tinbox translate --to de --model openai:gpt-5-2025-08-07 report.docx

# Save to file
tinbox translate --to es --output report_es.txt --model openai:gpt-5-2025-08-07 report.docx
```

## Model Providers

Tinbox supports multiple LLM providers:

<Tabs>
  <Tab title="OpenAI">
    ```bash
    # GPT-5 (latest)
    tinbox translate --to de --model openai:gpt-5-2025-08-07 document.txt
    
    # GPT-4o (vision-capable for PDFs)
    tinbox translate --to es --model openai:gpt-4o document.pdf
    ```
    
    <Note>
      GPT-4o and GPT-5 both support vision for PDF translation.
    </Note>
  </Tab>
  
  <Tab title="Anthropic (Claude)">
    ```bash
    # Claude 4 Sonnet (vision-capable)
    tinbox translate --to de --model anthropic:claude-3-sonnet document.pdf
    
    # For text documents
    tinbox translate --to fr --model anthropic:claude-3-sonnet document.txt
    ```
    
    Remember to set: `export ANTHROPIC_API_KEY="your-key"`
  </Tab>
  
  <Tab title="Google (Gemini)">
    ```bash
    # Gemini Pro Vision
    tinbox translate --to es --model google:gemini-pro-vision document.pdf
    
    # For text
    tinbox translate --to de --model google:gemini-pro document.txt
    ```
    
    Remember to set: `export GOOGLE_API_KEY="your-key"`
  </Tab>
  
  <Tab title="Ollama (Local)">
    ```bash
    # Start Ollama server first
    ollama serve  # In another terminal
    
    # Then use local models (free!)
    tinbox translate --to de --model ollama:llama3.1:8b document.txt
    tinbox translate --to fr --model ollama:mistral-small document.txt
    ```
    
    <Info>
      Ollama runs models locally - no API costs! But it doesn't support vision models for PDFs yet.
    </Info>
  </Tab>
</Tabs>

## Real-World Example

Let's translate a story from English to German with checkpointing:

<Steps>
  <Step title="Preview the Cost">
    Use `--dry-run` to estimate costs before translating:
    
    ```bash
    tinbox translate --to de --model openai:gpt-5-2025-08-07 --dry-run story.txt
    ```
    
    Output:
    ```
    📊 Translation Estimate:
    - Input tokens: ~1,200
    - Estimated output tokens: ~1,400
    - Estimated cost: $0.03
    - Estimated time: 15 seconds
    ```
  </Step>
  
  <Step title="Run with Checkpointing">
    For large documents, enable checkpointing to resume if interrupted:
    
    ```bash
    tinbox translate --to de \
      --model openai:gpt-5-2025-08-07 \
      --checkpoint-dir ./checkpoints \
      --output story_de.txt \
      story.txt
    ```
  </Step>
  
  <Step title="Monitor Progress">
    Tinbox shows real-time progress:
    
    ```
    🔄 Translating: story.txt
    📄 Processing chunk 1/8...
    💰 Cost so far: $0.01
    ⏱️  Elapsed: 5s
    
    ✅ Translation complete!
    📊 Final stats:
    - Total tokens: 2,600
    - Total cost: $0.03
    - Total time: 18s
    ```
  </Step>
</Steps>

## Advanced Features

### Glossary for Consistent Terminology

Maintain consistent translations of technical terms:

<CodeGroup>
  ```bash Auto-Generate Glossary
  # Tinbox will detect and save important terms
  tinbox translate --to es \
    --glossary \
    --save-glossary terms.json \
    --model openai:gpt-5-2025-08-07 \
    technical_doc.txt
  ```
  
  ```bash Use Existing Glossary
  # Use predefined terminology
  tinbox translate --to de \
    --glossary-file company_terms.json \
    --model openai:gpt-5-2025-08-07 \
    document.txt
  ```
  
  ```bash Extend Glossary
  # Start with existing terms and add new ones
  tinbox translate --to fr \
    --glossary-file base_terms.json \
    --save-glossary extended_terms.json \
    --model openai:gpt-5-2025-08-07 \
    document.txt
  ```
</CodeGroup>

#### Glossary File Format

Create a JSON file with term mappings:

```json terms.json
{
  "entries": {
    "API": "Interface de programmation",
    "CPU": "Processeur",
    "GPU": "Carte graphique",
    "Machine Learning": "Apprentissage automatique"
  }
}
```

### Cost Control

Prevent unexpected expenses:

```bash
# Set a maximum cost limit
tinbox translate --to de \
  --max-cost 5.00 \
  --model openai:gpt-5-2025-08-07 \
  document.txt

# Translation will stop if cost exceeds $5.00
```

### Reasoning Effort

Control translation quality vs. cost:

<CodeGroup>
  ```bash Minimal (Fast & Cheap)
  tinbox translate --to es \
    --reasoning-effort minimal \
    --model openai:gpt-5-2025-08-07 \
    document.txt
  ```
  
  ```bash Low (Balanced)
  tinbox translate --to de \
    --reasoning-effort low \
    --model openai:gpt-5-2025-08-07 \
    document.txt
  ```
  
  ```bash High (Best Quality)
  tinbox translate --to fr \
    --reasoning-effort high \
    --max-cost 10.00 \
    --model openai:gpt-5-2025-08-07 \
    document.txt
  ```
</CodeGroup>

<Tip>
  Start with `minimal` reasoning effort. Only increase to `low` or `high` if you need better quality for complex documents.
</Tip>

## Common Workflows

### Batch Translation

Translate multiple documents efficiently:

```bash
# Loop through all text files
for file in *.txt; do
  tinbox translate --to de \
    --model openai:gpt-5-2025-08-07 \
    --output "${file%.txt}_de.txt" \
    "$file"
done
```

### Resume Interrupted Translation

If a translation is interrupted, simply rerun with the same checkpoint directory:

```bash
# First run (interrupted)
tinbox translate --to de \
  --checkpoint-dir ./checkpoints \
  --model openai:gpt-5-2025-08-07 \
  large_document.txt

# Resume from checkpoint
tinbox translate --to de \
  --checkpoint-dir ./checkpoints \
  --model openai:gpt-5-2025-08-07 \
  large_document.txt
```

Tinbox automatically detects and resumes from the last checkpoint.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Translation quality is poor" icon="star-half">
    Try these improvements:
    1. Increase reasoning effort: `--reasoning-effort high`
    2. Use a better model: `openai:gpt-5-2025-08-07` or `anthropic:claude-3-sonnet`
    3. Enable glossary: `--glossary` for consistent terminology
    4. Specify source language: `--from en` (don't rely on auto-detect)
  </Accordion>
  
  <Accordion title="PDF translation not working" icon="file-pdf">
    Check these requirements:
    1. Poppler installed: Run `tinbox doctor` to verify
    2. Using vision-capable model: GPT-4o, Claude Sonnet, or Gemini Pro Vision
    3. PDF extras installed: `pip install tinbox[pdf]`
  </Accordion>
  
  <Accordion title="Too expensive" icon="dollar-sign">
    Reduce costs with:
    1. Use local Ollama models (free)
    2. Set `--reasoning-effort minimal`
    3. Set `--max-cost` limits
    4. Use `--dry-run` to preview costs
    5. Smaller context size: `--context-size 1500`
  </Accordion>
  
  <Accordion title="Translation timing out" icon="clock">
    For large documents:
    1. Enable checkpointing: `--checkpoint-dir ./checkpoints`
    2. Reduce chunk size: `--context-size 1500`
    3. For PDFs, use: `--algorithm page`
  </Accordion>
</AccordionGroup>
